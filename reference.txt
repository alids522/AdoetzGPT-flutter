Anda mengucapkan: openclaw apiOpenClaw exposes two primary API paths: the App SDK for connecting external applications, and the Gateway HTTP API for integrating tools and third-party frontends. OpenClaw's architecture operates as a self-hosted "Gateway" that bridges chat apps (like WhatsApp, Telegram, and Discord) to AI agents.1. The OpenClaw App SDKThe App SDK (available via @openclaw/sdk in Node.js) allows external scripts, CI jobs, IDE extensions, and dashboards to communicate directly with the OpenClaw Gateway. It lets you start agent runs, stream events, inspect transcripts, and manage tasks outside the core OpenClaw process.Usage Example:typescriptimport { OpenClaw } from '@openclaw/sdk';

const oc = new OpenClaw({ 
  url: "ws://127.0.0.1:18789", 
  token: process.env.OPENCLAW_GATEWAY_TOKEN,
  requestTimeoutMs: 30_000 
});

await oc.connect();

const agent = await oc.agents.get("main"); 
const run = await agent.run({ 
  input: "Review this pull request and suggest the smallest safe fix.", 
  model: "openai/gpt-5.5", 
  sessionKey: "main", 
  timeoutMs: 30_000,
});

for await (const event of run.events()) {
  // Handle streaming events (delta, tool calls, etc.)
  console.log(event.type);
}

const result = await run.wait({ timeoutMs: 120_000 });
console.log(result.status);

Instead, it works by passing the terminal execution capabilities of OpenClaw's own backend directly into the chat interface.Here is exactly how the integration works, how to configure it, and how to use it safely.How it Works Behind the ScenesWhen you connect OpenClaw to Open WebUI (either via an OpenAI-Compatible Connection or the OpenClaw Channels Plugin), Open WebUI behaves strictly as a frontend display.When you ask the AI to run a command:Open WebUI routes your prompt to the OpenClaw Gateway.OpenClaw processes the request and calls its built-in exec tool.The command runs on your underlying host machine or sandbox where the OpenClaw Gateway process is hosted.The terminal output streams directly back into your Open WebUI chat window.Step 1: Enable the Exec Tool in OpenClawBy default, executing shell commands is heavily guarded for security. Ensure your OpenClaw agent is configured to allow the exec tool in your openclaw.json5 or via the openclaw configure CLI wizard.You can set up your tools policy to allow execution, or enforce a strict allowlist / user-approval mechanism so it doesn't run arbitrary code without your consent:json5// openclaw.json5 snippet
{
  "tools": {
    "exec": {
      "enabled": true,
      "defaultHost": "sandbox" // Or "gateway" if running directly on the host machine
    }
  },
  "approvals": {
    "exec": {
      "mode": "ask" // Prompting you in the OpenClaw dashboard/logs before running terminal commands
    }
  }
}

Treat this as a direct remote code execution (RCE) surface for your application.Node.js / Fetch Examplejavascriptconst response = await fetch("http://127.0.0", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_GATEWAY_TOKEN"
  },
  body: JSON.stringify({
    tool: "exec", 
    args: {
      command: "echo 'Hello from my custom app!' && uname -a"
    }
  })
});

const data = await response.json();
if (data.ok) {
  console.log("Terminal Output:", data.result);
} else {
  console.error("Execution failed:", data.error.message);
}

const oc = new OpenClaw({ 
  url: "ws://127.0.0.1:18789", 
  token: process.env.OPENCLAW_GATEWAY_TOKEN 
});

await oc.connect();

// Invoke the exec tool directly through the policy pipeline
const envelope = await oc.tools.invoke({
  tool: "exec",
  args: {
    command: "ls -la /workspace"
  }
});

if (envelope.ok) {
  console.log("Success:", envelope.result);
} else {
  console.error("Error running command:", envelope.error);
}
